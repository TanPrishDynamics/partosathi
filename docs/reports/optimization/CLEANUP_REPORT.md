| Path | Size | Why safe to delete | How to restore |
|------|------|--------------------|----------------|
| `backend/venv/` | 558 MB | Local Python virtualenv — never committed (gitignored). Rebuilt from `requirements.txt`. | `python3.11 -m venv venv && venv/bin/pip install -r requirements.txt` |
| `backend/venv_production/` | 2.4 GB | Second local virtualenv carrying TensorFlow + Torch dylibs. Not referenced by any tooling. | n/a — not needed; was an experiment artifact |
| `lstm_pipeline/venv/` | 1.4 GB | Local virtualenv for the standalone ML pipeline. Rebuilt from `lstm_pipeline/requirements.txt`. | `python3 -m venv venv && pip install -r requirements.txt` |
| `frontend/node_modules/` | 182 MB | npm dependency tree — gitignored, reproducible. | `npm ci` |

**Sub-total reclaimed: ~4.65 GB**

> **Root cause of the bloat:** `backend/venv_production/` and a Finder-duplicated
> `lstm_pipeline/venv/lib 2/` directory each carried a full TensorFlow build
> (`libtensorflow_cc.2.dylib` is 589–659 MB *per copy*) plus PyTorch
> (`libtorch_cpu.dylib`, 165–207 MB). Three virtualenvs × heavy ML wheels = 4.3 GB.

## 6. Restore Instructions (Developer Onboarding)

After cloning or pulling this cleaned repo:

```bash
# Backend (use Python 3.11 — matches the production Docker image)
cd backend
python3.11 -m venv venv
venv/bin/pip install -r requirements.txt        # full install incl. torch
# (torch/pandas/numpy/scikit-learn require Python ≤3.12 wheels;
#  Python 3.14 lacks prebuilt wheels for the pinned versions)

# Clinician frontend
cd ../frontend && npm ci

# Admin frontend
cd ../admin-frontend && npm ci

# ML training pipeline (optional — not needed to run the API)
cd ../lstm_pipeline && python3 -m venv venv && venv/bin/pip install -r requirements.txt
```
